## v0.8.5 — 权限系统

### 权限模块

**新增 `src/lib/permission.ts`**：

- 通配符匹配：`*`（零或多个字符）、`?`（一个字符）、`**`（多级路径，仅 external_path）
- 最后匹配的规则优先
- 三级动作：`allow` / `ask` / `deny`
- 求值函数：`evaluate(permissionKey, input, rules)` → 返回 `allow` | `ask` | `deny`
- 手写通配符引擎，零依赖

**TypeScript 类型**：

```typescript
type PermissionAction = "allow" | "ask" | "deny";
type PermissionRules = PermissionAction | Record<string, PermissionAction>;

interface Permissions {
  external_path: PermissionRules;  // 粒度：文件路径
  execute: PermissionRules;        // 粒度：skill名:脚本名
  edit: PermissionRules;           // 粒度：文件路径（v0.8.5 预留，暂无内置 tool 消费）
}
```

**权限 key 与消费关系**：

| 权限 key | 粒度 | 默认 | v0.8.5 消费？ | 说明 |
|---|---|---|---|---|
| `external_path` | 文件路径通配符 | `"*": "ask"` | 是 | 内置 tool 读 workspace 外路径 |
| `execute` | `skill名:脚本名` 通配符 | `"*": "ask"` | 否（v0.9.0） | 执行 Skill 脚本，超级权限 |
| `edit` | 文件路径通配符 | `"*": "ask"` | 否（预留） | AI 编辑/写文件 |

**权限 key 之间的关系**：
- `external_path`、`edit`、`execute` 三者独立
- 内置 tool：检查 `external_path`（路径在 workspace 外时）+ `edit`（写操作时），各自独立求值
- `execute` tool：只检查 `execute` 权限，不再检查 `external_path` / `edit`（子进程无法拦截内部文件操作，允许执行 = 信任脚本全部行为）
- 信任模型：用户安装 Skill = 信任 Skill 作者，用户允许执行 = 信任脚本行为

**全局默认规则**：

```json
{
  "permissions": {
    "external_path": { "*": "ask" },
    "execute": { "*": "ask" },
    "edit": { "*": "ask" }
  }
}
```

**三级存储**：

| 层级 | 存储 | 生命周期 | 谁写入 |
|---|---|---|---|
| 全局规则 | `settings.json` 的 `permissions` | 永久 | Settings UI 手动配置 |
| session 规则 | 内存（按 chatKey 隔离） | 当前会话 | 内联确认条点"始终允许" |
| 仅本次 | 不存储 | 一次性 | 内联确认条点"允许" |

**求值顺序**：session 规则 > 全局规则 > 默认值

### `executeTool` 签名改造

**当前签名**：

```typescript
executeTool(name: string, args: Record<string, unknown>, signal: AbortSignal, ctx: ToolContext): Promise<string>
```

**改后**：

```typescript
executeTool(
  name: string,
  args: Record<string, unknown>,
  signal: AbortSignal,
  ctx: ToolContext,
  onPermission?: (request: PermissionRequest) => Promise<PermissionAction>
): Promise<string>
```

**设计决策**：选择方案 B（回调参数）而非方案 A（返回联合类型）或方案 C（提到 AISidebar 层）：
- 权限检查在 tool 内部最精确（知道具体路径、脚本名）
- 调用方（AISidebar）不需要关心权限逻辑
- OpenCode 也是每个 tool 内部调 `ctx.ask()`，同样模式

**`PermissionRequest` 类型**：

```typescript
interface PermissionRequest {
  permissionKey: "external_path" | "execute" | "edit";
  input: string;                    // 具体路径或 skill:script
  alwaysPatterns: string[];         // tool 建议的 always 通配模式
}
```

### 内置 tool 外部路径访问

**当前**：`isPathAllowed(path, workspaceRoot)` 硬拒绝 workspace 外路径

**改后**：
- workspace 内路径 → 直接允许（不变）
- workspace 外路径 → 调用 `evaluate("external_path", path, rules)`
  - `allow` → 放行
  - `ask` → 调用 `onPermission`，显示内联确认条
  - `deny` → 拒绝（返回错误给 AI）
- 没有 workspace 时（单文件模式）→ 保持当前行为（不触发外部路径检查）

**改动点**（两处 `isPathAllowed` 调用）：
- `src/lib/tools.ts:257` — `resolveContent()` 中（outline/read/read_section/grep 共用）
- `src/lib/tools.ts:467` — `toolLs()` 中

**Case 示例**：

用户在 workspace `~/projects/my-docs/` 中，让 AI 查看 `~/configs/nginx.conf`：

```
AI 调用 read({path: "~/configs/nginx.conf"})
  → resolveContent() 解析为 /home/user/configs/nginx.conf
  → workspace 外路径 → evaluate("external_path", "/home/user/configs/nginx.conf", rules)
  → session 规则无匹配，全局规则 "*": "ask" → ask
  → 调用 onPermission({ permissionKey: "external_path", input: "/home/user/configs/nginx.conf", alwaysPatterns: ["/home/user/configs/*"] })
  → AISidebar 显示确认条（输入框上方）
  → 用户点"始终允许"
    → sessionPermissions 新增: { permissionKey: "external_path", pattern: "/home/user/configs/*", action: "allow" }
    → 返回 "allow"
  → executeTool 继续执行

后续 AI 调用 read({path: "~/configs/app.conf"})
  → evaluate 匹配 session 规则 "/home/user/configs/*" → auto allow
  → 不显示确认条，直接放行

AI 调用 read({path: "/etc/hosts"})
  → session 规则 "/home/user/configs/*" 不匹配 "/etc/hosts"
  → 全局规则 "*": "ask" → 再次确认
```

### 内联确认条 UI 组件

**新增 `src/components/AISidebar/PermissionBar.tsx`**：

- 位置：AI 侧栏输入框上方
- 显示内容根据权限 key 不同：
  - `external_path`：`📂 访问 /path/to/file`
  - `execute`：`▶ 执行 script.py args...`
  - `edit`：`✏️ 编辑 /path/to/file`
- 三个按钮：
  - 允许 → 本次放行，不存储，返回 "allow"
  - 始终允许 → 生成通配规则写入 sessionPermissions，返回 "allow"
  - 拒绝 → 返回 "deny"
- 阻塞当前 tool 执行，等待用户响应
- 样式：Tailwind，与 AI 侧栏风格统一

**"始终允许"规则生成**：
- `external_path`：取目录级通配，如 `~/docs/*`（tool 的 `alwaysPatterns` 提供）
- `execute`：精确匹配 `skill名:脚本名`
- `edit`：取目录级通配，如 `~/docs/*`

### session 规则管理

**串行执行 + 级联自动通过**：

- tool 执行是串行的（for 循环 await）
- 第一个 tool 需要确认 → 用户选"始终允许" → 写入 session 规则
- 后续 tool 匹配同模式 → auto allow，不再确认
- 不同模式仍需确认
- 无需 pending map（串行不存在多个请求同时等待）

**存储**：

- 在哪个 store？v0.8.5 无 skillStore，可放在 chatStore 或新建 permissionStore
- 建议新增 `src/stores/permissionStore.ts`（轻量，只管 session 规则）

```typescript
interface PermissionState {
  sessionRules: Record<string, PermissionRule[]>;  // chatKey → 规则列表
  addSessionRule: (chatKey: string, rule: PermissionRule) => void;
  evaluateWithSession: (chatKey: string, key: string, input: string, globalRules: Permissions) => PermissionAction;
}

interface PermissionRule {
  permissionKey: string;
  pattern: string;
  action: PermissionAction;
}
```

## v0.9.0 — Skill 系统

### Skill 定义规范

**遵循 Agent Skills 开放标准**（agentskills.io）

**目录结构**：

```
{appDataDir}/skills/
  code-review/
    SKILL.md              # 必需：frontmatter + 指令
    scripts/              # 可选：可执行脚本
      lint_check.py
    references/           # 可选：参考文档
    assets/               # 可选：资源
```

**SKILL.md frontmatter**：

| 字段 | 必需 | 说明 |
|---|---|---|
| `name` | 是 | 1-64 字符，小写+数字+连字符，匹配目录名 |
| `description` | 是 | 1-1024 字符 |
| `license` | 否 | 许可证 |
| `compatibility` | 否 | 环境要求 |
| `metadata` | 否 | 任意 KV 对 |
| `allowed-tools` | 否 | 解析但不消费（对齐 OpenCode，实验性字段） |

**三层渐进加载**：

| 阶段 | 时机 | 读取内容 | 开销 |
|---|---|---|---|
| Discovery | 启动 | name + description | ~20 tokens/skill |
| Activation | AI 调用 skill({name}) | SKILL.md body | <5000 tokens |
| Execution | AI 调用 execute({skill, script}) | 执行 scripts/ 下脚本 | 取决于输出 |

**Skill 存放路径**：仅 `{appDataDir}/skills/<name>/`，不读项目目录

**改动文件**：
- 新增 `src/lib/skillParser.ts`：`parseSkillMd()`、`validateSkillName()`、`parseAllowedTools()`
- TypeScript 类型：`SkillManifest`、`Skill`

### Skill 发现

**新增 `src/lib/skillManager.ts`**：
- `discoverSkills(appDataDir)` → 扫描 `{appDataDir}/skills/*/SKILL.md`，解析 frontmatter
- 缓存结果，避免重复扫描

**新增 `src/stores/skillStore.ts`**（Zustand）：
- `skills: Skill[]` — 已发现的 skill 列表
- `activeSkillsMap: Record<string, string[]>` — chatKey → 已激活 skill names
- `sessionPermissions: Record<string, Record<string, Record<string, PermissionAction>>>` — chatKey → 权限覆盖
- `loadSkills()`、`toggleSkill(name)`、`activateSkill(chatKey, skillName)`

**启动时调用** `loadSkills()`（在 `initFromStartupData` 中）

### `skill` tool

**tool 定义**（`src/lib/tools.ts`）：

```typescript
{
  name: "skill",
  description: "Load a skill by name...\n\n<available_skills>...</available_skills>",
  parameters: {
    type: "object",
    properties: { name: { type: "string", description: "Skill name to load" } },
    required: ["name"]
  }
}
```

- description 中动态注入 enabled 的 skill 列表（name + description）
- AI 调用 `skill({name: "code-review"})` → 返回 SKILL.md body + 文件列表（ripgrep 或 readdir 列出最多 10 个文件）
- **权限：直接 allow，无需确认**

**执行逻辑**：
- 校验 name 是否在可用列表中
- 读取完整 SKILL.md body
- 列出 skill 目录下的文件（scripts/、references/、assets/）
- 返回 XML 格式内容（参考 OpenCode）

**改动文件**：
- `src/lib/tools.ts`：新增 `makeSkillTool()`、`executeTool` 新增 `"skill"` 分支

### `execute` tool

**tool 定义**：

```typescript
{
  name: "execute",
  description: "Execute a script from an activated skill's scripts directory",
  parameters: {
    type: "object",
    properties: {
      skill: { type: "string", description: "Skill name" },
      script: { type: "string", description: "Script filename" },
      args: { type: "array", items: { type: "string" }, description: "Script arguments" }
    },
    required: ["skill", "script"]
  }
}
```

**Rust 侧**（`src-tauri/src/lib.rs`）：

新增 `execute_skill_script` 命令：
- 校验：
  - skill 必须存在（`{appDataDir}/skills/{skill}/`）
  - script 路径 canonicalize 后必须以 `{skill_dir}/scripts/` 为前缀（防路径穿越）
  - skill 必须已激活（前端传参校验）
- 解释器匹配：`.py→python3` / `.js→node` / `.sh→bash` / `.ps1→powershell` / 其他→直接执行
- 执行：`Command::new(interpreter).arg(script_path).args(args).current_dir(workspace_root).output()`
- 环境变量：继承父进程环境（含 set_env_vars 设置的）
- 超时：30s（`tokio::time::timeout`）
- 返回：stdout 截断 30KB，非 0 退出码返回 stderr

**权限**：默认 ask，内联确认条
- `▶ 执行 lint_check.py src/main.ts [允许] [始终允许] [拒绝]`
- 求值 `evaluate("execute", "code-review:lint_check.py", rules)`

**改动文件**：
- `src/lib/tools.ts`：`executeTool` 新增 `"execute"` 分支
- `src-tauri/src/lib.rs`：新增 `execute_skill_script` 命令
- `src-tauri/tauri.conf.json`：权限允许新命令

## v0.9.5 — AI 提示词优化

### Selection AI 去掉全量文档

**当前**：`SelectionAIPanel.tsx` 中 explain/translate/rewrite 调用 `buildSystemPrompt(docContent, maxContext)` 发送整个文档

**改后**：轻量 system prompt `"You are the AI assistant for MoFlow editor. Help with the selected text."` + 精简 MD 语法块

**改动**：`src/components/Editor/SelectionAIPanel.tsx`、`src/i18n/locales/*.ts`

### Markdown 语法块精简

**当前**：~550 chars / ~138 tokens，12 种语法逐条列举（`ai.mdSyntax`）

**改后**：~200 chars / ~50 tokens，一句话概括

**改动**：`src/i18n/locales/en.ts`、`zh.ts`、`ja.ts`、`ko.ts`

### 工具说明去重

**当前**：system prompt 文本列工具 + tools 参数发 JSON Schema，重复

**改后**：system prompt 只保留一句引导，webfetch "Max 3 calls" 移入 tool description

**改动**：`src/lib/contextBuilder.ts`、`src/lib/tools.ts`、`src/i18n/locales/*.ts`

### Claude max_tokens 动态计算

**当前**：硬编码 `max_tokens: 4096`

**改后**：`Math.min(maxOutputTokens, maxContext - promptTokens)`，maxOutputTokens 来自 modelPricing.json

**改动**：`src/lib/llmClient.ts`、`src/lib/modelInfo.ts`、`src/lib/modelPricing.json`

### Token 估算改进

**当前**：fallback 只算 content 字段

**改后**：遍历 content + tool_calls[].arguments + reasoningContent + tool schema 固定开销

**改动**：`src/lib/llmClient.ts`

### translate 补 MD 格式提示

**当前**：translate prompt 不含 Markdown 语法提示

**改后**：追加精简版 MD 语法提示

**改动**：`src/i18n/locales/*.ts`

### API Token 改 type="password"

**改动**：`src/components/SettingsPanel/SettingsPanel.tsx`：API Token `<input type="text">` → `<input type="password">`
